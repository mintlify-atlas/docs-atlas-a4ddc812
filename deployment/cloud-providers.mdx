Deploy Mage on major cloud providers using managed container services for scalable, production-ready data pipelines. This guide covers AWS ECS, GCP Cloud Run, and Azure Container Instances.

## AWS Deployment

### AWS ECS with CloudFormation

Mage provides a complete CloudFormation template for deploying on AWS ECS Fargate with EFS storage and load balancing.

<Steps>
  <Step title="Prerequisites">
    - AWS account with appropriate permissions
    - AWS CLI configured
    - VPC with at least 2 subnets in different availability zones
  </Step>
  <Step title="Get VPC and Subnet IDs">
    ```bash
    aws ec2 describe-vpcs
    aws ec2 describe-subnets --filters "Name=vpc-id,Values=<your-vpc-id>"
    ```
  </Step>
  <Step title="Deploy with CloudFormation">
    ```bash
    aws cloudformation create-stack \
      --stack-name mage-data-preparation \
      --template-body file://ecs.yaml \
      --parameters \
        ParameterKey=VpcID,ParameterValue=vpc-xxxxx \
        ParameterKey=SubnetID,ParameterValue=subnet-xxxxx \
        ParameterKey=SubnetID2,ParameterValue=subnet-yyyyy \
      --capabilities CAPABILITY_NAMED_IAM
    ```
  </Step>
  <Step title="Monitor deployment">
    ```bash
    aws cloudformation describe-stacks --stack-name mage-data-preparation
    ```
    
    Get the Load Balancer URL from the stack outputs.
  </Step>
</Steps>

#### CloudFormation Template Overview

Here's the complete CloudFormation template from Mage's source:

```yaml ecs.yaml
AWSTemplateFormatVersion: "2010-09-09"
Parameters:
  SubnetID:
    Type: String
  SubnetID2:
    Type: String
  FileSharePort:
    Type: Number
    Description: Port to use for accessing the FileSystem via a Mount Target
    Default: 2049 # NFS Default Port Number
    MinValue: 1024
    MaxValue: 65535
  VpcID:
    Type: String

Resources:
  # ECS Cluster
  Cluster:
    Type: AWS::ECS::Cluster
    Properties:
      ClusterName: mage-data-preparation

  # CloudWatch Logs
  LogGroup:
    Type: AWS::Logs::LogGroup
    Properties:
      LogGroupName: /ecs/mage-data-preparation

  # IAM Role for ECS Task Execution
  ExecutionRole:
    Type: AWS::IAM::Role
    Properties:
      RoleName: mage-data-preparation
      AssumeRolePolicyDocument:
        Statement:
          - Effect: Allow
            Principal:
              Service: ecs-tasks.amazonaws.com
            Action: sts:AssumeRole
      ManagedPolicyArns:
        - arn:aws:iam::aws:policy/service-role/AmazonECSTaskExecutionRolePolicy

  # Security Group for Container
  ContainerSecurityGroup:
    Type: AWS::EC2::SecurityGroup
    Properties:
      GroupName: MageContainerSecurityGroup
      GroupDescription: Security group for Mage server
      SecurityGroupIngress:
        - IpProtocol: tcp
          FromPort: 6789
          ToPort: 6789
          CidrIp: 0.0.0.0/0

  # Elastic File System
  FileSystem:
    Type: AWS::EFS::FileSystem
    Properties:
      Encrypted: true
      FileSystemTags:
        - Key: Name
          Value: mage-data-preparation
      PerformanceMode: generalPurpose
      ThroughputMode: bursting

  MountTarget:
    Type: AWS::EFS::MountTarget
    Properties:
      FileSystemId: !GetAtt FileSystem.FileSystemId
      SubnetId: !Ref SubnetID
      SecurityGroups:
        - !GetAtt MountTargetSecurityGroup.GroupId

  MountTargetSecurityGroup:
    Type: AWS::EC2::SecurityGroup
    Properties:
      GroupDescription: FileSystem Security Group
      GroupName: MageEFSSecurityGroup
      SecurityGroupIngress:
        - IpProtocol: "tcp"
          FromPort: !Ref FileSharePort
          ToPort: !Ref FileSharePort
          SourceSecurityGroupId: !GetAtt ContainerSecurityGroup.GroupId

  # Load Balancer
  LoadBalancer:
    Type: AWS::ElasticLoadBalancingV2::LoadBalancer
    Properties:
      Name: mage-data-preparation
      Scheme: internet-facing
      LoadBalancerAttributes:
        - Key: idle_timeout.timeout_seconds
          Value: "30"
      Subnets:
        - !Ref SubnetID
        - !Ref SubnetID2
      SecurityGroups:
        - !GetAtt LoadBalancerSecurityGroup.GroupId

  LoadBalancerSecurityGroup:
    Type: AWS::EC2::SecurityGroup
    Properties:
      GroupDescription: Access to the public facing load balancer
      SecurityGroupIngress:
        - CidrIp: 0.0.0.0/0
          IpProtocol: -1

  TargetGroup:
    Type: AWS::ElasticLoadBalancingV2::TargetGroup
    Properties:
      VpcId: !Ref VpcID
      HealthCheckIntervalSeconds: 6
      HealthCheckPath: /api/kernels
      HealthCheckProtocol: HTTP
      HealthCheckTimeoutSeconds: 5
      HealthyThresholdCount: 2
      Port: 6789
      Protocol: HTTP
      TargetType: ip
      UnhealthyThresholdCount: 2

  LoadBalancerListener:
    Type: AWS::ElasticLoadBalancingV2::Listener
    Properties:
      DefaultActions:
        - Type: forward
          TargetGroupArn: !Ref TargetGroup
      LoadBalancerArn: !Ref LoadBalancer
      Port: 80
      Protocol: HTTP

  # ECS Task Definition
  TaskDefinition:
    Type: AWS::ECS::TaskDefinition
    Properties:
      Family: mage-data-preparation
      Cpu: 512
      Memory: 1024
      NetworkMode: awsvpc
      ExecutionRoleArn: !Ref ExecutionRole
      ContainerDefinitions:
        - Name: mage-data-preparation
          Image: mageai/mageai:latest
          MountPoints:
            - ReadOnly: false
              ContainerPath: /home/src
              SourceVolume: mage-data-preparation
          PortMappings:
            - ContainerPort: 6789
          PseudoTerminal: true
          LogConfiguration:
            LogDriver: awslogs
            Options:
              awslogs-region: !Ref AWS::Region
              awslogs-group: !Ref LogGroup
              awslogs-stream-prefix: ecs
      RequiresCompatibilities:
        - FARGATE
      Volumes:
        - Name: mage-data-preparation
          EfsVolumeConfiguration:
            FileSystemId: !GetAtt FileSystem.FileSystemId
            TransitEncryption: "ENABLED"

  # ECS Service
  Service:
    Type: AWS::ECS::Service
    DependsOn:
      - LoadBalancerListener
    Properties:
      ServiceName: mage-data-preparation
      Cluster: !Ref Cluster
      TaskDefinition: !Ref TaskDefinition
      DesiredCount: 1
      LaunchType: FARGATE
      LoadBalancers:
        - TargetGroupArn: !Ref TargetGroup
          ContainerName: mage-data-preparation
          ContainerPort: 6789
      NetworkConfiguration:
        AwsvpcConfiguration:
          AssignPublicIp: ENABLED
          Subnets:
            - !Ref SubnetID
          SecurityGroups:
            - !GetAtt ContainerSecurityGroup.GroupId
```

<Info>
  This template creates a complete ECS infrastructure with EFS for persistent storage, an Application Load Balancer, and CloudWatch logging.
</Info>

### AWS ECS Resource Details

The CloudFormation template provisions:

| Resource | Purpose |
| -------- | ------- |
| ECS Cluster | Container orchestration platform |
| ECS Task Definition | Container specification (512 CPU, 1024 MB memory) |
| ECS Service | Manages task lifecycle and desired count |
| EFS File System | Persistent storage for pipelines and data |
| Application Load Balancer | HTTP load balancing and health checks |
| IAM Execution Role | Permissions for ECS tasks |
| Security Groups | Network access control |
| CloudWatch Log Group | Container logs |

### GitHub Actions CI/CD for AWS ECS

Automate deployments with GitHub Actions:

```yaml .github/workflows/deploy-ecs.yml
name: Deploy to AWS ECS

on:
  push:
    branches:
      - main

env:
  AWS_REGION: us-east-1
  ECR_REPOSITORY: mage-app
  ECS_SERVICE: mage-data-preparation
  ECS_CLUSTER: mage-data-preparation
  ECS_TASK_DEFINITION: mage-data-preparation
  CONTAINER_NAME: mage-data-preparation

jobs:
  deploy:
    name: Deploy
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v3

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v2
        with:
          aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
          aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
          aws-region: ${{ env.AWS_REGION }}

      - name: Login to Amazon ECR
        id: login-ecr
        uses: aws-actions/amazon-ecr-login@v1

      - name: Build, tag, and push image to Amazon ECR
        id: build-image
        env:
          ECR_REGISTRY: ${{ steps.login-ecr.outputs.registry }}
          IMAGE_TAG: ${{ github.sha }}
        run: |
          docker build -t $ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG .
          docker push $ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG
          echo "image=$ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG" >> $GITHUB_OUTPUT

      - name: Download task definition
        run: |
          aws ecs describe-task-definition \
            --task-definition ${{ env.ECS_TASK_DEFINITION }} \
            --query taskDefinition > task-definition.json

      - name: Fill in the new image ID in the Amazon ECS task definition
        id: task-def
        uses: aws-actions/amazon-ecs-render-task-definition@v1
        with:
          task-definition: task-definition.json
          container-name: ${{ env.CONTAINER_NAME }}
          image: ${{ steps.build-image.outputs.image }}

      - name: Deploy Amazon ECS task definition
        uses: aws-actions/amazon-ecs-deploy-task-definition@v1
        with:
          task-definition: ${{ steps.task-def.outputs.task-definition }}
          service: ${{ env.ECS_SERVICE }}
          cluster: ${{ env.ECS_CLUSTER }}
          wait-for-service-stability: true
```

<Tip>
  Add secrets to your GitHub repository: `AWS_ACCESS_KEY_ID` and `AWS_SECRET_ACCESS_KEY`.
</Tip>

## GCP Deployment

### GCP Cloud Run

Deploy Mage on Google Cloud Run for serverless, auto-scaling container execution.

<Steps>
  <Step title="Set up GCP project">
    ```bash
    # Set project ID
    export PROJECT_ID=your-project-id
    export REGION=us-central1
    export SERVICE_NAME=mage-server
    
    gcloud config set project $PROJECT_ID
    ```
  </Step>
  <Step title="Enable required APIs">
    ```bash
    gcloud services enable run.googleapis.com
    gcloud services enable cloudbuild.googleapis.com
    gcloud services enable artifactregistry.googleapis.com
    ```
  </Step>
  <Step title="Build and push Docker image">
    ```bash
    # Create Artifact Registry repository
    gcloud artifacts repositories create mage-repo \
      --repository-format=docker \
      --location=$REGION
    
    # Build and push
    gcloud builds submit --tag $REGION-docker.pkg.dev/$PROJECT_ID/mage-repo/mage:latest
    ```
  </Step>
  <Step title="Deploy to Cloud Run">
    ```bash
    gcloud run deploy $SERVICE_NAME \
      --image $REGION-docker.pkg.dev/$PROJECT_ID/mage-repo/mage:latest \
      --platform managed \
      --region $REGION \
      --allow-unauthenticated \
      --port 6789 \
      --memory 2Gi \
      --cpu 2 \
      --timeout 3600 \
      --set-env-vars MAGE_DATA_DIR=/home/src/mage_data \
      --set-env-vars REQUIRE_USER_AUTHENTICATION=True
    ```
  </Step>
  <Step title="Get the service URL">
    ```bash
    gcloud run services describe $SERVICE_NAME --region $REGION --format 'value(status.url)'
    ```
  </Step>
</Steps>

<Warning>
  Cloud Run is stateless by default. For persistent storage, integrate with Cloud Storage or use Cloud SQL for PostgreSQL.
</Warning>

### GitHub Actions CI/CD for GCP Cloud Run

From Mage's source code:

```yaml .github/workflows/deploy-cloud-run.yml
name: Deploy to GCP Cloud Run

on:
  push:
    branches:
      - master

env:
  GAR_LOCATION: us-central1
  GOOGLE_CLOUD_RUN_SERVICE_NAME: mage-server
  IMAGE: mage
  PROJECT_ID: your-project-id
  REPOSITORY: mage-repo

jobs:
  build-and-deploy:
    name: Build, Publish, and Deploy
    runs-on: ubuntu-latest
    steps:
    - name: Checkout
      uses: actions/checkout@master

    - id: auth
      name: Authenticate to Google Cloud
      uses: google-github-actions/auth@v0.6.0
      with:
        credentials_json: ${{ secrets.GCP_CREDENTIALS }}
        token_format: access_token

    - uses: docker/login-action@v1
      name: Docker login
      with:
        registry: ${{ env.GAR_LOCATION }}-docker.pkg.dev
        username: oauth2accesstoken
        password: ${{ steps.auth.outputs.access_token }}

    - name: Docker build
      run: |-
        docker build \
          --tag "${{ env.GAR_LOCATION }}-docker.pkg.dev/${{ env.PROJECT_ID }}/${{ env.REPOSITORY }}/${{ env.IMAGE }}:${{ github.sha }}" \
          .

    - name: Docker push
      run: |-
        docker push "${{ env.GAR_LOCATION }}-docker.pkg.dev/${{ env.PROJECT_ID }}/${{ env.REPOSITORY }}/${{ env.IMAGE }}:${{ github.sha }}"

    - id: deploy
      uses: google-github-actions/deploy-cloudrun@v0
      with:
        service: ${{ env.GOOGLE_CLOUD_RUN_SERVICE_NAME }}
        image: ${{ env.GAR_LOCATION }}-docker.pkg.dev/${{ env.PROJECT_ID }}/${{ env.REPOSITORY }}/${{ env.IMAGE }}:${{ github.sha }}
        region: ${{ env.GAR_LOCATION }}
```

### GCP Cloud Storage Integration

For persistent storage, mount Cloud Storage:

```bash
# Install gcsfuse in custom Dockerfile
FROM mageai/mageai:latest

RUN apt-get update && apt-get install -y gcsfuse

# In Cloud Run, mount bucket
gcloud run deploy $SERVICE_NAME \
  --image $IMAGE_URL \
  --execution-environment gen2 \
  --update-env-vars GCS_BUCKET_NAME=your-bucket \
  --add-volume name=gcs,type=cloud-storage,bucket=your-bucket \
  --add-volume-mount volume=gcs,mount-path=/home/src
```

## Azure Deployment

### Azure Container Instances

Deploy Mage on Azure Container Instances for simple, serverless container hosting.

<Steps>
  <Step title="Create resource group">
    ```bash
    az group create --name mage-rg --location eastus
    ```
  </Step>
  <Step title="Create Azure Container Registry">
    ```bash
    az acr create --resource-group mage-rg --name mageregistry --sku Basic
    ```
  </Step>
  <Step title="Build and push image">
    ```bash
    # Login to ACR
    az acr login --name mageregistry
    
    # Build and push
    docker build -t mageregistry.azurecr.io/mage:latest .
    docker push mageregistry.azurecr.io/mage:latest
    ```
  </Step>
  <Step title="Deploy container instance">
    ```bash
    az container create \
      --resource-group mage-rg \
      --name mage-server \
      --image mageregistry.azurecr.io/mage:latest \
      --cpu 2 \
      --memory 4 \
      --registry-login-server mageregistry.azurecr.io \
      --registry-username $(az acr credential show --name mageregistry --query username -o tsv) \
      --registry-password $(az acr credential show --name mageregistry --query passwords[0].value -o tsv) \
      --ip-address Public \
      --ports 6789 \
      --environment-variables MAGE_DATA_DIR=/home/src/mage_data REQUIRE_USER_AUTHENTICATION=True
    ```
  </Step>
  <Step title="Get public IP">
    ```bash
    az container show --resource-group mage-rg --name mage-server --query ipAddress.ip -o tsv
    ```
  </Step>
</Steps>

### GitHub Actions CI/CD for Azure

From Mage's source code:

```yaml .github/workflows/deploy-azure.yml
name: Deploy to Azure Container Instance

on:
  push:
    branches:
      - master

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
    - name: 'Checkout GitHub Action'
      uses: actions/checkout@master
      
    - name: 'Login via Azure CLI'
      uses: azure/login@v1
      with:
        creds: ${{ secrets.AZURE_CREDENTIALS }}
    
    - uses: azure/docker-login@v1
      with:
        login-server: ${{ secrets.REGISTRY_LOGIN_SERVER }}
        username: ${{ secrets.REGISTRY_USERNAME }}
        password: ${{ secrets.REGISTRY_PASSWORD }}
    
    - run: |
        docker build . -t ${{ secrets.REGISTRY_LOGIN_SERVER }}/mageapp:${{ github.sha }}
        docker push ${{ secrets.REGISTRY_LOGIN_SERVER }}/mageapp:${{ github.sha }}

    - name: 'Deploy to Azure Container Instances'
      uses: 'azure/aci-deploy@v1'
      with:
        resource-group: ${{ secrets.RESOURCE_GROUP }}
        dns-name-label: ${{ secrets.RESOURCE_GROUP }}${{ github.run_number }}
        image: ${{ secrets.REGISTRY_LOGIN_SERVER }}/mageapp:${{ github.sha }}
        registry-login-server: ${{ secrets.REGISTRY_LOGIN_SERVER }}
        registry-username: ${{ secrets.REGISTRY_USERNAME }}
        registry-password: ${{ secrets.REGISTRY_PASSWORD }}
        name: mageapp-container
        location: 'west us'
```

### Azure Files for Persistent Storage

```bash
# Create storage account and file share
az storage account create \
  --resource-group mage-rg \
  --name magestorageacct \
  --location eastus \
  --sku Standard_LRS

az storage share create \
  --account-name magestorageacct \
  --name mage-data

# Deploy with volume mount
az container create \
  --resource-group mage-rg \
  --name mage-server \
  --image mageregistry.azurecr.io/mage:latest \
  --azure-file-volume-account-name magestorageacct \
  --azure-file-volume-account-key $(az storage account keys list --resource-group mage-rg --account-name magestorageacct --query [0].value -o tsv) \
  --azure-file-volume-share-name mage-data \
  --azure-file-volume-mount-path /home/src
```

## Cloud Provider Comparison

<Tabs>
  <Tab title="AWS">
    **Best for:** Enterprise workloads, existing AWS infrastructure
    
    **Pros:**
    - Mature ECS ecosystem
    - EFS for persistent storage
    - Extensive IAM controls
    - Load balancer integration
    
    **Cons:**
    - More complex setup
    - Higher minimum costs
    
    **Cost:** ~$50-100/month (Fargate + EFS + ALB)
  </Tab>
  <Tab title="GCP">
    **Best for:** Serverless, auto-scaling workloads
    
    **Pros:**
    - Simplest deployment
    - Auto-scaling to zero
    - Pay per request
    - Built-in HTTPS
    
    **Cons:**
    - Stateless by default
    - Cold start delays
    - Request timeout limits
    
    **Cost:** ~$20-50/month (Cloud Run)
  </Tab>
  <Tab title="Azure">
    **Best for:** Simple, low-cost deployments
    
    **Pros:**
    - Easy setup
    - Predictable pricing
    - Azure Files integration
    
    **Cons:**
    - Limited scaling options
    - No auto-scaling
    - Manual updates
    
    **Cost:** ~$30-60/month (ACI + Storage)
  </Tab>
</Tabs>

## Database Recommendations by Cloud

| Provider | Managed Database | Configuration |
| -------- | ---------------- | ------------- |
| AWS | RDS PostgreSQL | Use VPC peering for secure connection |
| GCP | Cloud SQL | Enable private IP and Cloud Run connector |
| Azure | Azure Database for PostgreSQL | Configure virtual network rules |
