### PostgreSQL

Extract data from PostgreSQL with support for logical replication.

**Configuration:**

```json
{
  "host": "localhost",
  "port": 5432,
  "database": "production",
  "schema": "public",
  "username": "mage_user",
  "password": "secure_password",
  "replication_slot": "mage_slot",
  "publication_name": "mage_pub"
}
```

**Replication Methods:**
- Full Table
- Incremental (based on timestamp/integer columns)
- Log-Based CDC (requires replication slot setup)

**Example:**

```python
from mage_integrations.sources.postgresql import PostgreSQL

source = PostgreSQL(
    config={
        'host': 'postgres.example.com',
        'port': 5432,
        'database': 'production',
        'schema': 'public',
        'username': 'readonly',
        'password': 'password123',
    }
)

# Discover tables
catalog = source.discover()

# Load data from specific tables
for records in source.load_data(stream, bookmarks={}):
    print(records)
```

<Accordion title="Setting up PostgreSQL CDC">
For log-based replication, configure a replication slot:

```sql
-- Create publication
CREATE PUBLICATION mage_pub FOR ALL TABLES;

-- Create replication slot
SELECT * FROM pg_create_logical_replication_slot('mage_slot', 'pgoutput');

-- Grant permissions
GRANT SELECT ON ALL TABLES IN SCHEMA public TO mage_user;
GRANT USAGE ON SCHEMA public TO mage_user;
```
</Accordion>

### Amazon S3

Read CSV and Parquet files from S3 buckets.

**Configuration:**

```json
{
  "aws_access_key_id": "AKIAIOSFODNN7EXAMPLE",
  "aws_secret_access_key": "wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY",
  "aws_region": "us-west-2",
  "bucket": "my-data-bucket",
  "prefix": "data/exports/",
  "search_pattern": ".*\\.csv$",
  "file_type": "csv"
}
```

**IAM Role Authentication:**

```json
{
  "role_arn": "arn:aws:iam::123456789012:role/MageDataRole",
  "role_session_name": "mage-data-integration",
  "aws_region": "us-west-2",
  "bucket": "my-data-bucket",
  "prefix": "exports/",
  "file_type": "parquet"
}
```

**Features:**
- Supports CSV and Parquet file formats
- Regex pattern matching for file selection
- Automatic schema inference
- Incremental sync based on file modification time
- Multiple stream configuration with `table_configs`
