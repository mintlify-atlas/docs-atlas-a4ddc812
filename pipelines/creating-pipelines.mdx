## Creating a Standard Pipeline

Standard (Python) pipelines are used for batch data processing workflows.

<Steps>
  <Step title="Create the pipeline">
    Use the `Pipeline.create()` class method:

    ```python
    from mage_ai.data_preparation.models.pipeline import Pipeline
    from mage_ai.data_preparation.models.constants import PipelineType

    pipeline = Pipeline.create(
        name='daily_etl_pipeline',
        repo_path='/home/src/my_project',
        pipeline_type=PipelineType.PYTHON,
        description='Daily ETL job to process customer data',
        tags=['production', 'daily', 'customer_data']
    )
    ```
  </Step>

  <Step title="Add blocks to the pipeline">
    ```python
    from mage_ai.data_preparation.models.block import Block
    from mage_ai.data_preparation.models.constants import BlockType

    # Create a data loader block
    loader = Block.create(
        name='load_customer_data',
        block_type=BlockType.DATA_LOADER,
        repo_path='/home/src/my_project',
        pipeline=pipeline
    )

    # Create a transformer block
    transformer = Block.create(
        name='clean_customer_data',
        block_type=BlockType.TRANSFORMER,
        repo_path='/home/src/my_project',
        pipeline=pipeline,
        upstream_block_uuids=['load_customer_data']
    )

    # Create a data exporter block
    exporter = Block.create(
        name='export_to_warehouse',
        block_type=BlockType.DATA_EXPORTER,
        repo_path='/home/src/my_project',
        pipeline=pipeline,
        upstream_block_uuids=['clean_customer_data']
    )
    ```
  </Step>

  <Step title="Configure the pipeline">
    ```python
    # Configure executor type
    await pipeline.update({
        'executor_type': 'local_python',
        'retry_config': {
            'retries': 3,
            'delay': 5,
            'exponential_backoff': True
        }
    })
    ```
  </Step>
</Steps>

## Creating a Streaming Pipeline

Streaming pipelines process data continuously from streaming sources.

```python
from mage_ai.data_preparation.models.pipeline import Pipeline
from mage_ai.data_preparation.models.constants import PipelineType, BlockType

# Create streaming pipeline
pipeline = Pipeline.create(
    name='kafka_stream_processor',
    repo_path='/home/src/my_project',
    pipeline_type=PipelineType.STREAMING,
    description='Process real-time events from Kafka'
)

# Add source block
source_block = Block.create(
    name='kafka_source',
    block_type=BlockType.DATA_LOADER,
    repo_path='/home/src/my_project',
    pipeline=pipeline,
    language='python'
)

# Add transformer
transform_block = Block.create(
    name='enrich_events',
    block_type=BlockType.TRANSFORMER,
    repo_path='/home/src/my_project',
    pipeline=pipeline,
    upstream_block_uuids=['kafka_source']
)

# Add sink block
sink_block = Block.create(
    name='write_to_database',
    block_type=BlockType.DATA_EXPORTER,
    repo_path='/home/src/my_project',
    pipeline=pipeline,
    upstream_block_uuids=['enrich_events']
)
```

### Streaming Pipeline Configuration

```yaml
# metadata.yaml for streaming pipeline
type: streaming
executor_count: 3  # Launch 3 parallel executors
executor_type: local_python
name: kafka_stream_processor

blocks:
- name: kafka_source
  type: data_loader
  language: python
  configuration:
    connector_type: kafka
    bootstrap_servers: localhost:9092
    topic: events
    group_id: mage_consumer
```

## Creating an Integration Pipeline

Integration pipelines sync data between sources and destinations.

<Tabs>
  <Tab title="Python API">
    ```python
    from mage_ai.data_preparation.models.pipeline import Pipeline
    from mage_ai.data_preparation.models.constants import PipelineType

    # Create integration pipeline
    pipeline = Pipeline.create(
        name='postgres_to_bigquery',
        repo_path='/home/src/my_project',
        pipeline_type=PipelineType.INTEGRATION,
        description='Sync PostgreSQL tables to BigQuery'
    )
    ```
  </Tab>

  <Tab title="YAML Configuration">
    ```yaml
    # Source configuration (YAML block)
    source: postgres
    config:
      host: localhost
      port: 5432
      database: production
      username: ${POSTGRES_USER}
      password: ${POSTGRES_PASSWORD}
    selected_streams:
    - users
    - orders
    - products

    # Destination configuration (YAML block)
    destination: bigquery
    config:
      project_id: my-gcp-project
      dataset: analytics
      credentials_json: ${BIGQUERY_CREDENTIALS}
    ```
  </Tab>
</Tabs>

### Integration Pipeline with Python Blocks

```python
# data_loader block for source
@data_loader
def load_from_postgres(*args, **kwargs):
    """
    Load data from PostgreSQL source.
    """
    return {
        'data_integration_uuid': 'postgres',
        'catalog': kwargs['catalog'],
        'config': {
            'host': 'localhost',
            'port': 5432,
            'database': 'production',
            'username': os.getenv('POSTGRES_USER'),
            'password': os.getenv('POSTGRES_PASSWORD'),
        },
        'selected_streams': ['users', 'orders'],
    }

# data_exporter block for destination
@data_exporter
def export_to_bigquery(data, *args, **kwargs):
    """
    Export data to BigQuery destination.
    """
    return {
        'data_integration_uuid': 'bigquery',
        'config': {
            'project_id': 'my-gcp-project',
            'dataset': 'analytics',
            'credentials_json': os.getenv('BIGQUERY_CREDENTIALS'),
        },
    }
```

## Creating a PySpark Pipeline

PySpark pipelines leverage Apache Spark for distributed data processing.

```python
from mage_ai.data_preparation.models.pipeline import Pipeline
from mage_ai.data_preparation.models.constants import (
    PipelineType,
    ExecutorType,
    BlockType
)

# Create PySpark pipeline
pipeline = Pipeline.create(
    name='large_scale_transformation',
    repo_path='/home/src/my_project',
    pipeline_type=PipelineType.PYSPARK,
    description='Process large datasets with Spark'
)

# Configure executor
await pipeline.update({
    'executor_type': ExecutorType.PYSPARK,
    'executor_config': {
        'spark_config': {
            'app_name': 'mage_pyspark_app',
            'spark_master': 'spark://spark-master:7077',
            'executor_memory': '4g',
            'driver_memory': '2g',
        }
    }
})
```

### PySpark Block Example

```python
from mage_ai.data_preparation.decorators import data_loader
from pyspark.sql import SparkSession

@data_loader
def load_data_from_s3(*args, **kwargs):
    """
    Load data from S3 using Spark.
    """
    spark = SparkSession.builder.getOrCreate()
    
    df = spark.read.parquet('s3a://my-bucket/data/')
    return df

@transformer
def transform_data(df, *args, **kwargs):
    """
    Transform data using Spark DataFrame operations.
    """
    from pyspark.sql import functions as F
    
    return df.filter(F.col('status') == 'active') \
             .groupBy('category') \
             .agg(F.sum('amount').alias('total_amount'))
```

### Retry Configuration

```python
# Configure retry behavior for failed blocks
await pipeline.update({
    'retry_config': {
        'retries': 3,                # Number of retry attempts
        'delay': 5,                  # Initial delay in seconds
        'max_delay': 60,             # Maximum delay between retries
        'exponential_backoff': True  # Use exponential backoff
    }
})
```

### Concurrency Configuration

```python
# Control concurrent block execution
await pipeline.update({
    'concurrency_config': {
        'block_run_limit': 5  # Max blocks running simultaneously
    }
})
```

## Duplicating Pipelines

Duplicate an existing pipeline:

```python
# Get source pipeline
source_pipeline = Pipeline.get('existing_pipeline', repo_path='/home/src/my_project')

# Duplicate it
duplicated = await Pipeline.duplicate(
    source_pipeline=source_pipeline,
    duplicate_pipeline_name='existing_pipeline_copy'
)

print(f"Created duplicate: {duplicated.uuid}")
```

## Best Practices

<Note>
  **Naming Conventions**
  
  - Use descriptive, lowercase names with underscores
  - Include the pipeline purpose: `daily_customer_etl`, `realtime_events_processor`
  - Avoid special characters except underscores and hyphens
</Note>

<Warning>
  **Pipeline UUIDs**
  
  Pipeline UUIDs are automatically generated from the name using `clean_name()`. Once created, changing the pipeline name will create a new UUID and effectively create a new pipeline.
</Warning>

<Tip>
  **Use Tags for Organization**
  
  ```python
  pipeline = Pipeline.create(
      name='my_pipeline',
      tags=['production', 'daily', 'critical', 'customer_facing']
  )
  ```
  
  Tags help filter and organize pipelines in the UI and API.
</Tip>
