## Overview

Mage supports real-time streaming pipelines that continuously process data from message queues, event streams, and other streaming sources. Streaming pipelines operate on micro-batches of data with configurable batch sizes and timeouts.

## Installation

Install Mage with streaming dependencies:

```bash
pip install mage-ai[streaming]
```

This includes support for (from setup.py:126-138):

- Kafka (kafka-python 2.3.0)

- Elasticsearch (elasticsearch 8.15.1)

- OpenSearch (opensearch-py 2.0.0)

### Kafka Sink Configuration

Write data to Kafka topics (kafka.py:62-141):

```python
from mage_ai.streaming.sinks.kafka import KafkaSink

config = {
    'bootstrap_server': 'localhost:9092',
    'topic': 'output_topic',
    'api_version': '0.10.2',
    'batch_size': 100,
    'timeout_ms': 500,
}

sink = KafkaSink(config)

# Write messages
messages = [
    {'data': {'user_id': 1, 'event': 'click'}},
    {'data': {'user_id': 2, 'event': 'view'}},
]

sink.batch_write(messages)
```

### Message Format

Mage supports two message formats (base.py:8):

1. **Simple format**: The entire message is the data
   ```python
   message = {'user_id': 1, 'event': 'click'}
   ```

2. **Metadata format**: Message contains data and metadata
   ```python
   message = {
       'data': {'user_id': 1, 'event': 'click'},
       'metadata': {
           'time': 1640000000000,  # timestamp in milliseconds
           'key': 'user_1',         # partition key
           'dest_topic': 'override_topic',  # override default topic
       }
   }
   ```

## Configuration Options

### Batch Processing

Configure batch size and timeout for optimal throughput (constants.py:3-4):

```python
config = {
    'batch_size': 100,      # Number of messages per batch (default: 100)
    'timeout_ms': 500,      # Max wait time in milliseconds (default: 500)
}
```

<Note>
  Larger batch sizes improve throughput but increase latency. Adjust based on your use case.
</Note>

### Buffer Management

Streaming sinks support buffering for fault tolerance (base.py:54-101):

```python
config = {
    'buffer_path': '/tmp/buffer.jsonl',  # Path to buffer file
    'buffer_timeout_seconds': 60,         # Buffer flush timeout
}
```

Messages are buffered to disk and automatically recovered on failure.

```python Data Loader (Kafka Source)
@streaming_source
def source(**kwargs) -> BaseSource:
    config = {
        'bootstrap_server': 'localhost:9092',
        'consumer_group': 'mage_group',
        'topic': 'user_events',
        'batch_size': 100,
        'timeout_ms': 500,
    }
    return KafkaSource(config)
```

```python Transformer
@transformer
def transform(messages, *args, **kwargs):
    """
    Process streaming messages in real-time
    """
    processed = []
    
    for message in messages:
        # Extract data from message
        data = message.get('data', message)
        
        # Transform the data
        processed.append({
            'user_id': data['user_id'],
            'event': data['event'],
            'processed_at': datetime.now().isoformat(),
        })
    
    return processed
```

## Performance Tuning

### Throughput Optimization

- Increase `batch_size` for higher throughput
- Increase `max_partition_fetch_bytes` for larger messages
- Use multiple consumer instances for parallelization

### Latency Optimization

- Decrease `timeout_ms` for lower latency
- Reduce `batch_size` for faster processing
- Optimize transformation logic

## Troubleshooting

### Consumer group rebalancing

Increase `session.timeout.ms` if consumers are being kicked out of the group.

### Message loss

Ensure `enable_auto_commit=False` and manually commit offsets after successful processing.

### High memory usage

Reduce `batch_size` and `max_partition_fetch_bytes` to limit memory consumption.
